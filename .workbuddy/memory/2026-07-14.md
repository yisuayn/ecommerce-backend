# 2026-07-14 工作日志

## 更新 API 文档

- 重写了 `API_DOCUMENT.md`，基于实际代码的完整接口文档（v2.0）
- 原有文档描述的是虚构模块（会员、订单、物流、CMS、财务、通知、营销、系统设置），实际代码中这些均不存在
- 实际项目包含以下模块及对应路由：

### 认证模块
- POST `/api/auth/register` - 注册
- POST `/api/auth/login` - 登录
- POST `/api/auth/logout` - 退出
- GET `/api/auth/me` - 获取当前用户（需认证）

### 分类管理
- GET `/api/getBaseCategoryList` - 树形分类
- POST/GET/PUT/DELETE `/api/category/:code` - CRUD
- GET `/api/category/:code/descendants` - 子孙分类

### 产品管理（前端）
- GET `/api/productlist` - 首页推荐
- GET `/api/seckill` - 秒杀
- GET `/api/getProductList` - 搜索/分页
- GET `/api/ProductDetail/:id` - 详情
- GET `/api/sku/:skuCode` - SKU查询

### 产品管理（后台）
- GET/POST/PUT/DELETE `/api/admin/:id` - CRUD
- POST/PUT `/api/admin/batch/delete`、`/api/admin/batch/status` - 批量操作
- PUT `/api/admin/:id/stock` - 库存更新
- GET `/api/admin/statistics` - 统计

### 品牌管理（前端）
- GET `/api/list` - 所有品牌
- GET `/api/hot` - 热门品牌
- GET `/api/detail/:id` - 品牌详情含产品

### 品牌管理（后台）
- GET/POST/PUT/DELETE `/api/admin` - CRUD
- POST `/api/admin/batch/delete` - 批量删除
- GET `/api/admin/statistics` - 统计

### 评论管理（前端）
- GET `/api/product/:productId` - 产品评价
- GET `/api/detail/:id` - 评价详情
- POST `/api/` - 创建评价
- POST `/api/:id/like` - 点赞
- POST `/api/:id/reply` - 商家回复

### 评论管理（后台）
- GET `/api/admin/list` - 管理列表
- GET `/api/admin/statistics` - 统计
- PUT `/api/admin/batch/status` - 批量审核
- POST `/api/admin/batch/delete` - 批量删除

### 轮播图
- GET `/api/getCarousel` - 前端轮播图（后台管理员接口已实现但未注册路由）

### 资讯
- GET `/api/getNewsInfo` - 分组新闻（后台管理接口已实现但未注册路由）

### 关键发现
1. **路由冲突**：产品/品牌/评论的后台管理接口路径重复（`/admin/list`、`/admin/statistics` 等），后注册的评论处理器覆盖了产品和品牌的处理器。文档中已标注并建议拆分路径。
2. **未注册接口**：轮播图和资讯的后台 CRUD 接口在 Controller/Service 层已完整实现，但未在 `routes/category.js` 中注册路由。
3. 数据库为 SQL Server（表名 `chengyue`），使用 Sequelize ORM 连接。
